POST /v1/users/

The user will be sent a confirmation email that they'll need to click on before they can publish events.

Request payload

attribute type description
email_address string (required) The user's email address
password string (required) The user's password

Response payload

attribute type description
token string The user's auth token.

PATCH /v1/users/me

Modify a user

Request payload

attribute type description
confirmation_token string Confirmation token sent to a user's email address

PATCH /v1/users/me/duo

Modify a user's duo enrollment

Request payload

attribute type description
status string One of "on", "off", or "enrolling"

Response payload

attribute type description
activation.barcode url Display this url to the end user so they can scan the qr code
activation.code url Launch this url on android or ios in order to launch the duo app
activation.expiration rfc3339 encoded date Expiration date of the enrollment request
status string One of "on", "off", or "enrolling"
